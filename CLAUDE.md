# Project conventions

## Git workflow

- **Branch naming**: always work on a `feature/<slug>` branch. Never commit directly to `main`.
  - Before starting any change, create or switch to an appropriate `feature/` branch.
  - The slug should be short and descriptive in English (e.g. `feature/eval-event-codes`).
- **Commit messages**: write in English, follow Conventional Commits (`fix:`, `feat:`, `refactor:`, `chore:`, `docs:`).
  - Do NOT include a `Co-Authored-By` trailer in any commit.
- **Pull requests**: title and body must be written in English.

## Language

All code, comments, commit messages, PR titles/bodies, and documentation must be written in English.
